### Schema Markup to Add

#### Organization Schema
```json
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "PoolApp",
  "applicationCategory": "BusinessApplication",
  "operatingSystem": "iOS, Android, Web",
  "offers": {
    "@type": "Offer",
    "price": "59.00",
    "priceCurrency": "USD",
    "priceValidUntil": "2026-12-31"
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.9",
    "ratingCount": "127"
  }
}
```

#### FAQ Schema (for FAQ sections)
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "How much does pool service software cost?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "PoolApp starts at $59/month with no per-technician fees..."
      }
    }
  ]
}
```

### URL Structure Best Practices
- Use hyphens, not underscores
- Keep URLs short and keyword-focused
- Follow logical hierarchy
- No duplicate content across URLs

## Conversion Optimization + SEO Balance

### Keep What's Working
- Urgency messaging for campaigns (but make it dynamic/removable)
- Strong social proof elements
- Clear pricing comparison
- Before/after feature presentations

### Add for SEO
- More text content (search engines need words)
- H2 sections with keyword variations
- FAQ sections with schema
- Comparison tables with competitor names
- Internal links throughout

### Testing Recommendations
1. A/B test headline with keyword vs. emotional hook
2. Test long-form vs. short-form landing pages
3. Track organic keyword rankings monthly
4. Monitor organic traffic vs. conversion rate trade-off
